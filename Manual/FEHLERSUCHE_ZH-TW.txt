===================================================================================
  Evi Portable — 擴充故障排除（參考）
  補充手冊\README.txt（概述）和模型\llm\README.txt
===================================================================================

本指南涵蓋典型和不太常見的問題：原因、檢查、修復。
從上到下工作或跳到匹配部分。

注意：此處並非每個螢幕訊息都逐字引用 - 請使用組合
症狀（你看到了什麼？）和章節標題。

------------------------------------------------------------------------------------------------
  快速檢查（始終是第一位）
------------------------------------------------------------------------------------------------

  □ Windows 10 或 11（64 位元），目前更新，重大變更後重新啟動？
  □ Evi 從完全解壓縮的資料夾開始 — 不是半解壓縮的存檔，
    而不是僅從雲端（同步資料夾）“實時”？
  □models\llm\ 中至少有一個有效的.gguf（請參閱models\llm\README.txt）？
  □ 足夠的可用磁碟空間（模型+聊天+更新）？
  □更改設定、備忘單或模型後：完全退出 Evi
    然後重新開始？


------------------------------------------------------------------------------------------------
  1. 作業系統基礎知識
------------------------------------------------------------------------------------------------

症狀：安裝或啟動錯誤，“不支援......”
說明：Evi 僅針對 Windows 10/11。其他系統不受支援。

修復：
  • 僅使用Windows 10 或11。
  • 通常不支援 32 位元 Windows — 使用 64 位元。
  • 安裝Windows 更新，重新啟動PC。

症狀：雙擊應用程式沒有任何反應
說明：政策阻止、缺少權限或啟動程序位於受阻止的路徑中。

修復：
  • 僅在需要時以滑鼠右鍵按一下→ 以管理員身分執行（正常情況：標準使用者）。
  • 請勿從ZIP 內部運作－完全解壓縮。
  • 檢查防毒軟體是否記錄了封鎖（請參閱第 3 節）。


------------------------------------------------------------------------------------------------
  2. 資料夾路徑、可攜性、雲和防毒
------------------------------------------------------------------------------------------------

症狀：檔案「消失」、資料夾重複、.gguf 損壞
說明：雲端同步（OneDrive、Dropbox等）可以鎖定檔案或上傳部分文件
  Evi 寫入時的資料。

修復：
  • 將 Evi 資料夾放在本機磁碟機上，例如C:\EviPortable 或 D:\Tools\Evi —
    不是“Documents\OneDrive\...”，對即時資料進行主動同步。
  • 為 Evi 根新增安全軟體排除項目（掃描每個小
    寫入可能會減慢或嚴重鎖定 LLM/Piper）。

症狀：USB 安裝 — 速度非常慢或出現錯誤
解釋：速度慢、FAT32 大小限制、連線不良。

修復：
  • USB 3.x 記憶棒，NTFS，足夠的空間；喜歡內建SSD。
  • 不要在非常慢的介質上運行大型模型－使用較小的GGUF。

症狀：「路徑太長」或特殊字元錯誤
說明：Windows MAX_PATH / Unicode — 罕見但深的路徑 + 奇怪的字符
  破壞一些工具。

修復：
  • 使用短路徑，例如C:\Evi\，如果可能，請避免路徑中出現非 ASCII。


------------------------------------------------------------------------------------------------
  3. 語言模型 (.gguf) — 無回應、模型錯誤、掛起
------------------------------------------------------------------------------------------------

症狀：“No AI”，空答案，模型無法加載
解釋：預期資料夾中沒有有效的 GGUF，下載不完整，錯誤
  格式或損壞的檔案。

修復：
  • 開啟模型\llm\：至少一個具有合理大小（不是0 位元組）的.gguf。
  • 重新下載；在瀏覽器中檢查名稱中是否有奇怪的副檔名 - 重新命名
    僅當您知道文件完整時。
  • 閱讀models\llm\README.txt：建議的Qwen-2.5 檔案和VRAM 層。
  • 多個GGUF：根據自述文件，可能存在多個文件— Evi 選擇符合VRAM。
    如有疑問，請僅在資料夾中保留一個已知良好的檔案。

症狀：儘管有 GPU，但反應速度非常慢
解釋：模型對於 GPU → 卸載/CPU 來說太大，或預設錯誤。

修復：
  • 較小的模型（例如 7B q4_k_m 而不是 14B）。
  • 將側欄中的GPU 預設與真實硬體進行比對。
症狀：CUDA / GPU 錯誤、「記憶體不足」、生成時崩潰
解釋：VRAM 不足，另一個程序使用 GPU，筆記型電腦卡在 iGPU 上。

修復：
  • 更新NVIDIA 驅動程式(nvidia.com)。
  • 在筆記型電腦上，BIOS/Windows 中為 Evi 強制使用“獨立 GPU”/“高效能”
    如果有的話。
  • 嘗試Evi 中的CPU 預設（緩慢但穩定）。
  • 關閉其他GPU 密集型應用程式（遊戲、其他AI 工具）。


------------------------------------------------------------------------------------------------
  4. GPU、驅動程式、筆記型電腦混合顯示卡
------------------------------------------------------------------------------------------------

症狀：雖然安裝了 NVIDIA GPU，但 Evi 卻看不到 NVIDIA GPU
解釋：電源模式錯誤、缺少驅動程式、混合動力始終使用 iGPU。

修復：
  • 裝置管理員：NVIDIA 沒有錯誤代碼？
  • 「高效能」/「終極」模式（供應商應用程式：Lenovo Vantage 等）。
  • 驅動程式更新後重新啟動。

症狀：螢幕閃爍、驅動程式崩潰（TDR）
說明：圖形逾時 — 通常是 GPU 負載過多或超頻不穩定。

修復：
  • 測試期間禁止超頻；較小的型號；更好的冷卻。


------------------------------------------------------------------------------------------------
  5. 僅 CPU－期望與限制
------------------------------------------------------------------------------------------------

症狀：“這需要永遠”
說明：7B+ 時的 CPU 推理比 GPU 慢得多 — 這是正常的。

修復：
  • 每個models\llm\README.txt 保留足夠的RAM。
  • 關閉後台作業；電源計畫「高效能」。


------------------------------------------------------------------------------------------------
  6. 啟動、許可證、機器 ID
------------------------------------------------------------------------------------------------

症狀：序號不被接受
解釋：拼字錯誤、空格、不同硬體的金鑰、過時的 ID。

修復：
  • 小心貼上金鑰，不要有前導/尾隨空格。
  • 確保郵件與此PC/有效的機器ID 相符。
  • 更換主機板/CPU 後：新ID — 聯絡支援人員並提供證明鏈。

現象：沒有回覆郵件
說明：垃圾郵件資料夾、寄件者地址錯誤、主題錯誤。

修復：
  • 檢查垃圾郵件；重新傳送並明確主題和機器 ID。


------------------------------------------------------------------------------------------------
  7. 麥克風 — 沒有電平，Evi 聽不到任何聲音
------------------------------------------------------------------------------------------------

症狀：無捕獲，儀表空
解釋：Windows 阻止存取、錯誤的輸入裝置、電纜/USB。

修復：
  • 設定→ 隱私權→ 麥克風→ 允許桌面應用程式。
  • 聲音設定→ 輸入：設定並測試正確的預設值（「語音」）。
  • 在Evi 中：選擇麥克風，儲存，重新啟動應用程式。
  • 藍牙耳機：配對、電池；延遲可能很糟糕——嘗試使用 USB 麥克風。

症狀：麥克風僅在某些應用程式中有效
說明：另一個應用程式的獨佔模式阻止了裝置。

修復：
  • 關閉其他通訊應用程式或停用「專用麥克風」。


------------------------------------------------------------------------------------------------
  8.語音辨識（本地）
------------------------------------------------------------------------------------------------

症狀：語音無法識別，處理“卡住”
解釋：Whisper 模型遺失/無效、首次下載、CPU/GPU 繁忙。

修復：
  • 只有在必須取得模型時才使用網路 - 請參閱 models\whisper。
  • 簡短地打字而不是說話來隔離Evi。
  • 主要更新後：重新啟動Evi。


------------------------------------------------------------------------------------------------
  9. 語音輸出（TTS/Piper）－無聲或錯誤的語言
------------------------------------------------------------------------------------------------

症狀：雖然顯示文字回答但沒有聲音
說明：沒有語言語音包、語音輸入錯誤、Windows 音量。

修復：
  • 檢查 Piper\ — 是否存在所選語言的聲音？
  • 在Evi 中選擇與UI 語言相符的語音，然後儲存。
  • Windows 音量混合器：Evi 未靜音？
  • 嘗試其他輸出裝置（耳機與顯示器）。


------------------------------------------------------------------------------------------------
 10.網路、防火牆、“封鎖所有流量”
------------------------------------------------------------------------------------------------

症狀：網路、郵件助理、天氣、下載失敗
說明：Evi、Windows 防火牆、企業代理程式、VPN 中的全域封鎖。

修復：
  • 在 Evi 中，如果有線上功能，請關閉「封鎖所有流量」（或同等功能）
    被通緝。
  • 暫時放鬆VPN/防火牆（僅在允許的情況下）。
  • 公司網路：向IT 詢問代理例外情況。


------------------------------------------------------------------------------------------------
 11. YouTube、媒體、VLC 便攜式
------------------------------------------------------------------------------------------------

症狀：影片無法啟動，“缺少組件”
解釋：VLC Portable 遺失、網路阻塞、路徑錯誤。

修復：
  • 請勿刪除Portable\VLCPortable（或提供的佈局）。
  • 允許網路（請參閱第10 節）。


------------------------------------------------------------------------------------------------
 12. UI 語言、備忘單
------------------------------------------------------------------------------------------------

症狀：命令不起作用，文字舊
解釋：備忘單已編輯，但 Evi 未重新啟動；檔案名稱錯誤。

修復：
  • 閱讀cheatsheets\README.txt — 每個語言代碼一個檔案（例如de.txt、en.txt）。
  • 手動編輯後完全重新啟動Evi。
  • 將檔案另存為UTF-8（重音）。


------------------------------------------------------------------------------------------------
 13. 安全性：PIN 碼和語音解鎖
------------------------------------------------------------------------------------------------

症狀：忘記 PIN 碼，語音解鎖總是失敗
說明：安全功能－故意難以繞過；房間太吵了。

修復：
  • 房間安靜，麥克風水平，沒有其他人重疊的聲音。
  • PIN/個人恢復僅根據供應商政策－提供證據支持。
  • 切勿分享您的PIN 碼。


------------------------------------------------------------------------------------------------
 14. 效能、崩潰、日誌
------------------------------------------------------------------------------------------------

症狀：凍結、突然崩潰
解釋：VRAM、RAM、熱節流、壞檔案、更新後的錯誤。

修復：
  • 較小的型號；關閉其他應用程式；檢查冷卻。
  • 檢查Evi 資料夾中的crash.log 或類似檔案（如果存在）。
  • 撤銷上次變更（刪除新模型、復原設定）。

症狀：一切都感覺遲緩
說明：後台服務、完整 SSD、非常大的模型情境。

修復：
  • 工作管理員：CPU/RAM/GPU；電源計畫；可用 SSD 空間（約 10-15% 為空）。
