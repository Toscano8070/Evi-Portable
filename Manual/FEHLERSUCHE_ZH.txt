  Evi Portable — 扩展故障排除（参考）
  补充手册\README.txt（概述）和模型\llm\README.txt

本指南涵盖典型和不太常见的问题：原因、检查、修复。
从上到下工作或跳到匹配部分。

注意：此处并非每条屏幕消息都逐字引用 - 请使用组合
症状（你看到了什么？）和章节标题。

------------------------------------------------------------------------------------------------
  快速检查（始终是第一位）
------------------------------------------------------------------------------------------------

  □ Windows 10 或 11（64 位），当前更新，重大更改后重新启动？
  □ Evi 从完全解压的文件夹开始 — 不是半解压的存档，
    而不是仅从云（同步文件夹）“实时”？
  □ models\llm\ 中至少有一个有效的.gguf（请参阅models\llm\README.txt）？
  □ 足够的可用磁盘空间（模型+聊天+更新）？
  □ 更改设置、备忘单或模型后：完全退出 Evi
    然后重新开始？


------------------------------------------------------------------------------------------------
  1. 操作系统基础知识
------------------------------------------------------------------------------------------------

症状：安装或启动错误，“不支持……”
说明：Evi 仅针对 Windows 10/11。其他系统不受支持。

修复：
  • 仅使用Windows 10 或11。
  • 通常不支持 32 位 Windows — 使用 64 位。
  • 安装Windows 更新，重新启动PC。

症状：双击应用程序没有任何反应
说明：策略阻止、缺少权限或启动程序位于受阻止的路径中。

修复：
  • 仅在需要时右键单击→ 以管理员身份运行（正常情况：标准用户）。
  • 不要从ZIP 内部运行——完全解压。
  • 检查防病毒软件是否记录了阻止（请参阅第 3 节）。


------------------------------------------------------------------------------------------------
  2. 文件夹路径、可移植性、云和防病毒
------------------------------------------------------------------------------------------------

症状：文件“消失”、文件夹重复、.gguf 损坏
说明：云同步（OneDrive、Dropbox等）可以锁定文件或上传部分文件
  Evi 写入时的数据。

修复：
  • 将 Evi 文件夹放在本地驱动器上，例如C:\EviPortable 或 D:\Tools\Evi —
    不是“Documents\OneDrive\...”，对实时数据进行主动同步。
  • 为 Evi 根添加安全软件排除项（扫描每个小
    写入可能会减慢或严重锁定 LLM/Piper）。

症状：USB 安装 — 速度非常慢或出现错误
解释：速度慢、FAT32 大小限制、连接不良。

修复：
  • USB 3.x 记忆棒，NTFS，足够的空间；更喜欢内置SSD。
  • 不要在非常慢的介质上运行大型模型——使用较小的GGUF。

症状：“路径太长”或特殊字符错误
说明：Windows MAX_PATH / Unicode — 罕见但深的路径 + 奇怪的字符
  破坏一些工具。

修复：
  • 使用短路径，例如C:\Evi\，如果可能，请避免路径中出现非 ASCII。


------------------------------------------------------------------------------------------------
  3. 语言模型 (.gguf) — 无回复、模型错误、挂起
------------------------------------------------------------------------------------------------

症状：“No AI”，空答案，模型无法加载
解释：预期文件夹中没有有效的 GGUF，下载不完整，错误
  格式或损坏的文件。

修复：
  • 打开模型\llm\：至少一个具有合理大小（不是0 字节）的.gguf。
  • 重新下载；在浏览器中检查名称中是否有奇怪的扩展名 - 重命名
    仅当您知道文件完整时。
  • 阅读models\llm\README.txt：推荐的Qwen-2.5 文件和VRAM 层。
  • 多个GGUF：根据自述文件，可能存在多个文件— Evi 选择匹配VRAM。
    如有疑问，请仅在文件夹中保留一个已知良好的文件。

症状：尽管有 GPU，但响应速度非常慢
解释：模型对于 GPU → 卸载/CPU 来说太大，或预设错误。

修复：
  • 较小的模型（例如 7B q4_k_m 而不是 14B）。
  • 将侧栏中的GPU 预设与真实硬件进行匹配。
症状：CUDA / GPU 错误、“内存不足”、生成时崩溃
解释：VRAM 不足，另一个进程使用 GPU，笔记本电脑卡在 iGPU 上。

修复：
  • 更新NVIDIA 驱动程序(nvidia.com)。
  • 在笔记本电脑上，BIOS/Windows 中为 Evi 强制使用“独立 GPU”/“高性能”
    如果有的话。
  • 尝试Evi 中的CPU 预设（缓慢但稳定）。
  • 关闭其他GPU 密集型应用程序（游戏、其他AI 工具）。


------------------------------------------------------------------------------------------------
  4. GPU、驱动程序、笔记本电脑混合显卡
------------------------------------------------------------------------------------------------

症状：虽然安装了 NVIDIA GPU，但 Evi 却看不到 NVIDIA GPU
解释：电源模式错误、缺少驱动程序、混合动力始终使用 iGPU。

修复：
  • 设备管理器：NVIDIA 没有错误代码？
  • “高性能”/“终极”模式（供应商应用程序：Lenovo Vantage 等）。
  • 驱动程序更新后重新启动。

症状：屏幕闪烁、驱动程序崩溃（TDR）
说明：图形超时 — 通常是 GPU 负载过多或超频不稳定。

修复：
  • 测试期间禁止超频；较小的型号；更好的冷却。


------------------------------------------------------------------------------------------------
  5. 仅 CPU——期望和限制
------------------------------------------------------------------------------------------------

症状：“这需要永远”
说明：7B+ 时的 CPU 推理比 GPU 慢得多 — 这是正常的。

修复：
  • 每个models\llm\README.txt 保留足够的RAM。
  • 关闭后台作业；电源计划“高性能”。


------------------------------------------------------------------------------------------------
  6. 激活、许可证、机器 ID
------------------------------------------------------------------------------------------------

症状：序列号不被接受
解释：拼写错误、空格、不同硬件的密钥、过时的 ID。

修复：
  • 小心粘贴密钥，不要有前导/尾随空格。
  • 确保邮件与此PC/有效的机器ID 匹配。
  • 更换主板/CPU 后：新ID — 联系支持人员并提供证明链。

现象：没有回复邮件
说明：垃圾邮件文件夹、发件人地址错误、主题错误。

修复：
  • 检查垃圾邮件；重新发送并明确主题和机器 ID。


------------------------------------------------------------------------------------------------
  7. 麦克风 — 没有电平，Evi 听不到任何声音
------------------------------------------------------------------------------------------------

症状：无捕获，仪表空
解释：Windows 阻止访问、错误的输入设备、电缆/USB。

修复：
  • 设置→ 隐私→ 麦克风→ 允许桌面应用程序。
  • 声音设置→ 输入：设置并测试正确的默认值（“语音”）。
  • 在Evi 中：选择麦克风，保存，重新启动应用程序。
  • 蓝牙耳机：配对、电池；延迟可能很糟糕——尝试使用 USB 麦克风。

症状：麦克风仅在某些应用程序中有效
说明：另一个应用程序的独占模式阻止了设备。

修复：
  • 关闭其他通讯应用程序或禁用“专用麦克风”。


------------------------------------------------------------------------------------------------
  8.语音识别（本地）
------------------------------------------------------------------------------------------------

症状：语音无法识别，处理“卡住”
解释：Whisper 模型丢失/无效、首次下载、CPU/GPU 繁忙。

修复：
  • 仅当必须获取模型时才使用网络 - 请参阅 models\whisper。
  • 简短地打字而不是说话来隔离Evi。
  • 主要更新后：重新启动Evi。


------------------------------------------------------------------------------------------------
  9. 语音输出（TTS/Piper）——无声或错误的语言
------------------------------------------------------------------------------------------------

症状：虽然显示文字回答但没有声音
说明：没有语言语音包、语音输入错误、Windows 音量。

修复：
  • 检查 Piper\ — 是否存在所选语言的声音？
  • 在Evi 中选择与UI 语言匹配的语音，然后保存。
  • Windows 音量混合器：Evi 未静音？
  • 尝试其他输出设备（耳机与显示器）。


------------------------------------------------------------------------------------------------
 10.网络、防火墙、“阻止所有流量”
------------------------------------------------------------------------------------------------

症状：网络、邮件助手、天气、下载失败
说明：Evi、Windows 防火墙、企业代理、VPN 中的全局阻止。

修复：
  • 在 Evi 中，如果有在线功能，请关闭“阻止所有流量”（或同等功能）
    被通缉。
  • 暂时放松VPN/防火墙（仅在允许的情况下）。
  • 公司网络：向IT 询问代理例外情况。


------------------------------------------------------------------------------------------------
 11. YouTube、媒体、VLC 便携式
------------------------------------------------------------------------------------------------

症状：视频无法启动，“缺少组件”
解释：VLC Portable 丢失、网络阻塞、路径错误。

修复：
  • 不要删除Portable\VLCPortable（或提供的布局）。
  • 允许网络（参见第10 节）。


------------------------------------------------------------------------------------------------
 12. UI 语言、备忘单
------------------------------------------------------------------------------------------------

症状：命令不起作用，文本旧
解释：备忘单已编辑，但 Evi 未重新启动；文件名错误。

修复：
  • 阅读cheatsheets\README.txt — 每个语言代码一个文件（例如de.txt、en.txt）。
  • 手动编辑后完全重新启动Evi。
  • 将文件另存为UTF-8（重音）。


------------------------------------------------------------------------------------------------
 13. 安全性：PIN 码和语音解锁
------------------------------------------------------------------------------------------------

症状：忘记 PIN 码，语音解锁总是失败
说明：安全功能——故意难以绕过；房间太吵了。

修复：
  • 房间安静，麦克风水平，没有其他人重叠的声音。
  • PIN/个人恢复仅根据供应商政策——提供证据支持。
  • 切勿分享您的PIN 码。


------------------------------------------------------------------------------------------------
 14. 性能、崩溃、日志
------------------------------------------------------------------------------------------------

症状：冻结、突然崩溃
解释：VRAM、RAM、热节流、坏文件、更新后的错误。

修复：
  • 较小的型号；关闭其他应用程序；检查冷却。
  • 检查Evi 文件夹中的crash.log 或类似文件（如果存在）。
  • 撤消上次更改（删除新模型、恢复设置）。

症状：一切都感觉迟缓
说明：后台服务、完整 SSD、非常大的模型上下文。

修复：
  • 任务管理器：CPU/RAM/GPU；电源计划；可用 SSD 空间（大约 10-15% 为空）。
