============================================================================
  Evi Portable - 확장된 문제 해결(참조)
  보충 설명서\README.txt(개요) 및 models\llm\README.txt
============================================================================

이 가이드에서는 원인, 확인, 수정 등 일반적인 문제와 덜 일반적인 문제를 다룹니다.
위에서 아래로 작업하거나 일치하는 섹션으로 이동하세요.

참고: 화면에 표시되는 모든 메시지가 여기에 그대로 인용된 것은 아닙니다. 다음 조합을 사용하십시오.
증상(무엇이 보이나요?) 및 섹션 제목.

--------------------------------------------------------------------------------
  빠른 확인(항상 먼저)
--------------------------------------------------------------------------------

  □ Windows 10 또는 11(64비트), 현재 업데이트, 주요 변경 후 재부팅됩니까?
  □ Evi는 반쯤 압축을 푼 아카이브가 아닌 완전히 추출된 폴더에서 시작되었습니다.
    클라우드(동기화 폴더)에서만 "라이브"되지 않습니까?
  □ models\llm\에 유효한 .gguf가 하나 이상 있습니다(models\llm\README.txt 참조).
  □ 여유 디스크 공간(모델 + 채팅 + 업데이트)이 충분합니까?
  □ 설정, 치트 시트, 모델 변경 후: Evi를 완전히 종료합니다.
    그리고 다시 시작해?


--------------------------------------------------------------------------------
  1. 운영 체제 기본 사항
--------------------------------------------------------------------------------

증상: 설치 또는 시작 오류, "...에서 지원되지 않음"
설명: Evi는 Windows 10/11만을 대상으로 합니다. 다른 시스템은 지원되지 않습니다.

수정 사항:
  • Windows 10 또는 11만 사용하세요.
  • 32비트 Windows는 일반적으로 지원되지 않습니다. 64비트를 사용하십시오.
  • Windows 업데이트를 설치하고 PC를 다시 시작합니다.

증상: 앱을 두 번 클릭해도 아무 반응이 없습니다.
설명: 정책 차단, 권한 누락 또는 차단된 경로의 실행 프로그램입니다.

수정 사항:
  • 마우스 오른쪽 버튼 클릭 → 필요한 경우에만 관리자 권한으로 실행합니다(일반적인 경우: 표준 사용자).
  • ZIP 내부에서 실행하지 마십시오. 완전히 추출하십시오.
  • 바이러스 백신이 차단을 기록했는지 확인합니다(섹션 3 참조).


--------------------------------------------------------------------------------
  2. 폴더 경로, 이식성, 클라우드 및 바이러스 백신
--------------------------------------------------------------------------------

증상: 파일이 "사라지고", 폴더가 중복되고, .gguf가 손상되었습니다.
설명: 클라우드 동기화(OneDrive, Dropbox 등)는 파일을 잠그거나 부분적으로 업로드할 수 있습니다.
  Evi가 쓰는 동안 데이터.

수정 사항:
  • Evi 폴더를 로컬 드라이브에 넣습니다. C:\EviPortable 또는 D:\Tools\Evi —
    라이브 데이터에 대한 활성 동기화를 사용하는 "Documents\OneDrive\..."가 아닙니다.
  • Evi 루트에 대한 보안 소프트웨어 제외를 추가합니다(모든 작은 항목을 검사).
    쓰기는 LLM/Piper를 느리게 하거나 잠길 수 있습니다).

증상: USB 설치 - 매우 느리거나 오류가 발생함
설명: 느린 스틱, FAT32 크기 제한, 연결 불량.

수정 사항:
  • USB 3.x 스틱, NTFS, 충분한 공간; 내장 SSD를 선호합니다.
  • 매우 느린 미디어에서는 거대한 모델을 실행하지 말고 더 작은 GGUF를 사용하십시오.

증상: "경로가 너무 깁니다" 또는 특수 문자 오류
설명: Windows MAX_PATH / 유니코드 — 드물지만 깊은 경로 + 이상한 문자
  일부 도구를 부수십시오.

수정 사항:
  • 짧은 경로를 사용하세요. C:\Evi\, 가능하면 경로에 ASCII가 아닌 문자는 사용하지 마세요.


--------------------------------------------------------------------------------
  3. 언어 모델(.gguf) - 응답 없음, 모델 오류, 정지
--------------------------------------------------------------------------------

증상: "AI 없음", 빈 답변, 모델이 로드되지 않음
설명: 예상 폴더에 유효한 GGUF가 없습니다. 다운로드가 불완전하며 잘못되었습니다.
  형식이거나 파일이 손상되었습니다.

수정 사항:
  • Open models\llm\: 적절한 크기(0바이트 아님)를 가진 하나 이상의 .gguf.
  • 다시 다운로드; 브라우저에서 이름에 이상한 확장자가 있는지 확인하십시오. — 이름 바꾸기
    파일이 완전하다는 것을 알고 있는 경우에만 가능합니다.
  • 모델\llm\README.txt 읽기: 권장되는 Qwen-2.5 파일 및 VRAM 계층입니다.
  • 여러 GGUF: README에 따라 여러 파일이 존재할 수 있습니다. Evi는 VRAM과 일치하도록 선택합니다.
    의심스러우면 폴더에 정상 작동이 확인된 파일 하나만 남겨두십시오.

증상: GPU에도 불구하고 응답 속도가 매우 느림
설명: GPU → 오프로드/CPU에 비해 모델이 너무 크거나 사전 설정이 잘못되었습니다.

수정 사항:
  • 더 작은 모델(예: 14B 대신 7B q4_k_m).
  • 사이드바의 GPU 사전 설정을 실제 하드웨어와 일치시킵니다.
증상: CUDA/GPU 오류, "메모리 부족", 생성 중 충돌
설명: VRAM이 충분하지 않고, 다른 프로세스가 GPU를 사용하고, 노트북이 iGPU에 붙어 있습니다.

수정 사항:
  • NVIDIA 드라이버를 업데이트하십시오(nvidia.com).
  • 노트북의 경우 BIOS/Windows에서 Evi에 대해 "개별 GPU"/"고성능"을 강제 적용합니다.
    가능한 곳.
  • Evi에서 CPU 사전 설정을 사용해 보세요(느리지만 안정적임).
  • GPU를 많이 사용하는 다른 앱(게임, 기타 AI 도구)을 닫습니다.


--------------------------------------------------------------------------------
  4. GPU, 드라이버, 노트북 하이브리드 그래픽
--------------------------------------------------------------------------------

증상: NVIDIA GPU가 설치되어 있어도 Evi에 NVIDIA GPU가 표시되지 않습니다.
설명: 잘못된 전원 모드, 드라이버 누락, 하이브리드가 항상 iGPU에 있습니다.

수정 사항:
  • 장치 관리자: 오류 코드가 없는 NVIDIA?
  • “고성능” / “Ultimate” 모드(공급업체 앱: Lenovo Vantage, …).
  • 드라이버 업데이트 후 재부팅하세요.

증상: 화면 깜박임, 드라이버 충돌(TDR)
설명: 그래픽 시간 초과 - GPU 로드가 너무 많거나 오버클럭이 불안정한 경우가 많습니다.

수정 사항:
  • 테스트 중에는 오버클럭이 불가능합니다. 더 작은 모델; 더 나은 냉각.


--------------------------------------------------------------------------------
  5. CPU 전용 - 기대치와 한계
--------------------------------------------------------------------------------

증상: “시간이 오래 걸립니다”
설명: 7B+의 CPU 추론은 GPU보다 훨씬 느립니다. 이는 정상입니다.

수정 사항:
  • 모델\llm\README.txt별로 충분한 RAM을 유지하십시오.
  • 백그라운드 작업을 닫습니다. 전원 계획 "고성능".


--------------------------------------------------------------------------------
  6. 활성화, 라이선스, 컴퓨터 ID
--------------------------------------------------------------------------------

증상: 일련번호가 허용되지 않음
설명: 오타, 공백, 다른 하드웨어의 키, 오래된 ID입니다.

수정 사항:
  • 선행/후행 공백 없이 키를 조심스럽게 붙여넣습니다.
  • 메일이 이 PC/유효한 컴퓨터 ID와 일치하는지 확인하세요.
  • 메인보드/CPU 교체 후: 새 ID — 증명 체인을 통해 지원팀에 문의하세요.

증상: 회신 메일이 없습니다.
설명: 스팸 폴더, 보낸 사람 주소가 잘못되었습니다. 제목이 잘못되었습니다.

수정 사항:
  • 스팸을 확인하세요. 명확한 제목과 컴퓨터 ID를 사용하여 다시 보내세요.


--------------------------------------------------------------------------------
  7. 마이크 - 레벨 없음, Evi는 아무 소리도 듣지 못함
--------------------------------------------------------------------------------

증상: 캡처가 되지 않고 미터가 비어 있습니다.
설명: Windows가 액세스를 차단합니다. 잘못된 입력 장치, 케이블/USB입니다.

수정 사항:
  • 설정 → 개인정보 보호 → 마이크 → 데스크톱 앱을 허용합니다.
  • 사운드 설정 → 입력: 올바른 기본값(“음성”)을 설정하고 테스트합니다.
  • Evi에서: 마이크를 선택하고 저장한 후 앱을 다시 시작합니다.
  • 블루투스 헤드셋: 페어링, 배터리; 대기 시간이 나쁠 수 있습니다. USB 마이크를 사용해 보세요.

증상: 마이크가 일부 앱에서만 작동함
설명: 다른 앱의 독점 모드로 인해 기기가 차단되었습니다.

수정 사항:
  • 다른 통신 앱을 닫거나 '전용 마이크'를 비활성화하세요.


--------------------------------------------------------------------------------
  8. 음성인식(로컬)
--------------------------------------------------------------------------------

증상: 음성이 인식되지 않고 처리가 "멈췄습니다"
설명: Whisper 모델 누락/잘못됨, 최초 다운로드, CPU/GPU 사용 중.

수정 사항:
  • 모델을 가져와야 하는 경우에만 네트워크에 연결합니다. 모델\속삭임을 참조하세요.
  • Evi를 분리하기 위해 말하는 대신 짧게 입력하세요.
  • 주요 업데이트 후: Evi를 다시 시작합니다.


--------------------------------------------------------------------------------
  9. 음성 출력(TTS/Piper) — 조용하거나 잘못된 언어
--------------------------------------------------------------------------------

증상: 텍스트 답변이 표시되지만 소리가 들리지 않음
설명: 언어에 대한 음성 팩이 없음, 잘못된 음성 입력, Windows 볼륨.

수정 사항:
  • 파이퍼 확인\ — 선택한 언어에 대한 음성이 있습니까?
  • Evi에서 음성 일치 UI 언어를 선택하고 저장합니다.
  • Windows 볼륨 믹서: Evi가 음소거되지 않았나요?
  • 다른 출력 장치(헤드폰 대 모니터)를 사용해 보십시오.


--------------------------------------------------------------------------------
 10. 네트워크, 방화벽, “모든 트래픽 차단”
--------------------------------------------------------------------------------

증상: 웹, 메일 도우미, 날씨, 다운로드 실패
설명: Evi, Windows 방화벽, 기업 프록시, VPN의 전역 차단입니다.

수정 사항:
  • Evi에서 온라인 기능이 있는 경우 "모든 트래픽 차단"(또는 이에 상응하는 기능)을 끄십시오.
    수배됩니다.
  • VPN/방화벽을 일시적으로 완화합니다(허용되는 경우에만).
  • 기업 네트워크: IT 부서에 프록시 예외를 요청하세요.


--------------------------------------------------------------------------------
 11. 유튜브, 미디어, VLC 포터블
--------------------------------------------------------------------------------

증상: 비디오가 시작되지 않고 "구성 요소 누락"이 발생합니다.
설명: VLC 휴대용 누락, 네트워크 차단, 잘못된 경로.

수정 사항:
  • Portable\VLCPortable(또는 제공된 레이아웃)을 삭제하지 마십시오.
  • 네트워크를 허용합니다(섹션 10 참조).


--------------------------------------------------------------------------------
 12. UI 언어, 치트 시트
--------------------------------------------------------------------------------

증상: 명령이 작동하지 않습니다. 오래된 텍스트입니다.
설명: 치트 시트가 편집되었지만 Evi가 다시 시작되지 않았습니다. 파일 이름이 잘못되었습니다.

수정 사항:
  • 치트시트\README.txt 읽기 — 언어 코드당 파일 1개(예: de.txt, en.txt).
  • 수동 편집 후 Evi를 완전히 다시 시작합니다.
  • 파일을 UTF-8(악센트)로 저장합니다.


--------------------------------------------------------------------------------
 13. 보안: PIN 및 음성 잠금 해제
--------------------------------------------------------------------------------

증상: PIN 분실, 음성 잠금 해제가 항상 실패함
설명: 보안 기능 - 의도적으로 우회하기 어렵습니다. 방이 너무 시끄러워요.

수정 사항:
  • 조용한 방, 마이크 수준, 다른 사람의 목소리가 겹치지 않습니다.
  • 공급업체 정책에 따라 PIN/개인 복구만 가능 - 증거 지원.
  • PIN을 절대 공유하지 마십시오.


--------------------------------------------------------------------------------
 14. 성능, 충돌, 로그
--------------------------------------------------------------------------------

증상: 정지, 갑작스러운 충돌
설명: VRAM, RAM, 열 조절 장치, 잘못된 파일, 업데이트 후 버그.

수정 사항:
  • 더 작은 모델; 다른 앱을 닫습니다. 냉각을 확인하십시오.
  • Evi 폴더에 crash.log 또는 유사한 항목이 있는 경우 이를 확인하십시오.
  • 마지막 변경을 취소합니다(새 모델 제거, 설정 되돌리기).

증상: 모든 것이 느리게 느껴진다
설명: 백그라운드 서비스, 전체 SSD, 매우 큰 모델 컨텍스트.

수정 사항:
  • 작업 관리자: CPU/RAM/GPU; 전원 계획; 여유 SSD 공간(약 10~15% 비어 있음)


--------------------------------------------------------------------------------
 15. 아무것도 도움이 되지 않는 경우 - 지원 메일 정보

  • 컴퓨터 ID(활성화/정보 화면)

  • .gguf(파일 이름, 대략적인 크기)
  • 내가 수행한 짧은 단계
  • 정확한 오류 텍스트 또는 스크린샷(개인 키를 표시하지 마세요!)
