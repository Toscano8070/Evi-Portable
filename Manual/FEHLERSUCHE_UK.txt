===================================================================================
  Evi Portable — Розширене усунення несправностей (довідка)
  Додатки Manual\README.txt (огляд) і моделі\llm\README.txt
===================================================================================

Цей посібник охоплює типові та менш поширені проблеми: причини, перевірки, виправлення.
Працюйте зверху вниз або перейдіть до відповідного розділу.

Примітка: не кожне повідомлення на екрані цитується тут дослівно — використовуйте комбінацію
ознаки (що ви бачите?) і заголовок розділу.

--------------------------------------------------------------------------------
  Швидка перевірка (завжди спочатку)
--------------------------------------------------------------------------------

  □ Windows 10 або 11 (64-розрядна), поточні оновлення, перезавантаження після серйозних змін?
  □ Еві розпочала роботу з повністю розпакованої папки, а не з напіврозпакованого архіву,
    а не «живий» лише з хмари (папка синхронізації)?
  □ Принаймні один дійсний .gguf у models\llm\ (див. models\llm\README.txt)?
  □ Чи достатньо вільного місця на диску (модель + чати + оновлення)?
  □ Після змін налаштувань, шпаргалок або моделей: повністю вийдіть з Evi
    і почати знову?


--------------------------------------------------------------------------------
  1. Основи операційної системи
--------------------------------------------------------------------------------

Симптом: помилки встановлення або запуску, «не підтримується на…»
Пояснення: Evi призначено лише для Windows 10/11. Інші системи не підтримуються.

Виправлення:
  • Використовуйте лише Windows 10 або 11.
  • 32-розрядна Windows зазвичай не підтримується — використовуйте 64-розрядну.
  • Встановити оновлення Windows, перезавантажити ПК.

Симптом: подвійне клацання програми нічого не робить
Пояснення: блокування політики, відсутні дозволи або програма запуску на заблокованому шляху.

Виправлення:
  • Клацніть правою кнопкою миші → Запускати від імені адміністратора лише за потреби (звичайний випадок: стандартний користувач).
  • Не запускайте з ZIP — повністю розпакуйте.
  • Перевірити, чи зареєстрував антивірус блокування (див. розділ 3).


--------------------------------------------------------------------------------
  2. Шляхи до папок, портативність, хмара та антивірус
--------------------------------------------------------------------------------

Симптом: файли «зникають», дублікати папок, пошкоджений .gguf
Пояснення: хмарна синхронізація (OneDrive, Dropbox, …) може блокувати файли або завантажувати їх частково
  даних, поки Еві пише.

Виправлення:
  • Помістіть папку Evi на локальний диск, напр. C:\EviPortable або D:\Tools\Evi —
    а не «Документи\OneDrive\…» з активною синхронізацією поточних даних.
  • Додайте виключення програмного забезпечення безпеки для кореня Evi (сканування кожного дрібного
    запис може уповільнити або погано заблокувати LLM/Piper).

Симптом: інсталяція USB — дуже повільна або помилки
Пояснення: повільне зависання, обмеження розміру FAT32, погане з’єднання.

Виправлення:
  • флешка USB 3.x, NTFS, достатньо місця; віддайте перевагу внутрішньому SSD.
  • Не запускайте величезні моделі на дуже повільних носіях — використовуйте менший GGUF.

Симптом: «Задовгий шлях» або помилки спеціальних символів
Пояснення: Windows MAX_PATH / Unicode — рідкісні, але глибокі шляхи + непарні символи
  зламати деякі інструменти.

Виправлення:
  • Використовуйте короткий шлях, напр. C:\Evi\, уникайте не-ASCII у шляху, якщо можливо.


--------------------------------------------------------------------------------
  3. Мовна модель (.gguf) — немає відповідей, помилки моделі, зависання
--------------------------------------------------------------------------------

Симптом: «Немає ШІ», пусті відповіді, модель не завантажується
Пояснення: немає дійсного GGUF в очікуваній папці, неповне завантаження, неправильно
  формат або пошкоджений файл.

Виправлення:
  • Відкриті моделі\llm\: принаймні один .gguf з правдоподібним розміром (не 0 байтів).
  • Повторне завантаження; в браузері перевірте наявність дивних розширень в імені — перейменуйте
    лише якщо ви знаєте, що файл повний.
  • Читайте models\llm\README.txt: рекомендовані файли Qwen-2.5 і рівні VRAM.
  • Кілька GGUF: для README може існувати кілька файлів — Evi вибирає відповідно до VRAM.
    Якщо сумніваєтеся, залиште в папці лише ОДИН завідомо справний файл.

Симптом: Відповідає дуже повільно, незважаючи на GPU
Пояснення: модель завелика для графічного процесора → розвантаження/ЦП, або неправильне налаштування.

Виправлення:
  • Менша модель (наприклад, 7B q4_k_m замість 14B).
  • Зіставте налаштування GPU на бічній панелі з реальним обладнанням.
Симптом: помилка CUDA / GPU, «брак пам’яті», збій під час генерації
Пояснення: недостатньо відеопам’яті, інший процес використовує GPU, ноутбук застряг на iGPU.

Виправлення:
  • Оновіть драйвер NVIDIA (nvidia.com).
  • На ноутбуках у BIOS/Windows примусово примусово «discrete GPU» / «high performance» для Evi
    де доступно.
  • Спробуйте попередньо налаштований ЦП у Evi (повільно, але стабільно).
  • Закрийте інші програми, що навантажують GPU (ігри, інші інструменти AI).


--------------------------------------------------------------------------------
  4. Графічний процесор, драйвери, гібридна графіка ноутбука
--------------------------------------------------------------------------------

Симптом: Evi не бачить графічний процесор NVIDIA, хоча його встановлено
Пояснення: неправильний режим живлення, відсутній драйвер, гібрид завжди на iGPU.

Виправлення:
  • Диспетчер пристроїв: NVIDIA без коду помилки?
  • Режим «Висока продуктивність» / «Максимальний» (програми постачальників: Lenovo Vantage, …).
  • Перезавантаження після оновлення драйвера.

Симптом: мерехтіння екрана, збій драйвера (TDR)
Пояснення: тайм-аут графіки — часто занадто велике навантаження на GPU або нестабільний розгін.

Виправлення:
  • Відсутність розгону під час тестів; менша модель; краще охолодження.


--------------------------------------------------------------------------------
  5. Лише процесор — очікування та обмеження
--------------------------------------------------------------------------------

Симптом: «Це займає вічність»
Пояснення: визначення ЦП на рівні 7B+ набагато повільніше, ніж ГП — це нормально.

Виправлення:
  • Зберігайте достатньо оперативної пам’яті для models\llm\README.txt.
  • Закрити фонові завдання; план живлення «Висока продуктивність».


--------------------------------------------------------------------------------
  6. Активація, ліцензія, ID машини
--------------------------------------------------------------------------------

Симптом: серійний номер не приймається
Пояснення: помилки, пробіли, ключ для іншого обладнання, застарілий ідентифікатор.

Виправлення:
  • Обережно вставляйте ключ, без пробілів на початку та в кінці.
  • Переконайтеся, що пошта відповідає цьому ПК/дійсному ідентифікатору машини.
  • Після заміни материнської плати/ЦП: новий ідентифікатор — зверніться до служби підтримки з ланцюжком доказів.

Симптом: немає відповіді на лист
Пояснення: папка зі спамом, неправильна адреса відправника, неправильна тема.

Виправлення:
  • Перевірити спам; повторно надіслати з чіткою темою та ідентифікатором машини.


--------------------------------------------------------------------------------
  7. Мікрофон — немає рівня, Еві нічого не чує
--------------------------------------------------------------------------------

Симптом: немає захоплення, лічильник порожній
Пояснення: Windows блокує доступ, неправильний пристрій введення, кабель/USB.

Виправлення:
  • Налаштування → Конфіденційність → Мікрофон → дозволити настільні програми.
  • Параметри звуку → Введення: установіть і перевірте правильне значення за замовчуванням («мовлення»).
  • В Evi: вибрати мікрофон, зберегти, перезапустити програму.
  • Bluetooth-гарнітура: сполучення, акумулятор; затримка може бути поганою — спробуйте USB-мікрофон.

Симптом: мікрофон працює лише в деяких програмах
Пояснення: ексклюзивний режим іншої програми блокує пристрій.

Виправлення:
  • Закрийте інші комунікаційні програми або вимкніть «ексклюзивний мікрофон».


--------------------------------------------------------------------------------
  8. Розпізнавання мовлення (локальне)
--------------------------------------------------------------------------------

Симптом: мовлення не розпізнається, «застрягла» обробка
Пояснення: модель Whisper відсутня/недійсна, перше завантаження, CPU/GPU зайнятий.

Виправлення:
  • Мережа, лише якщо потрібно отримати моделі — див. models\whisper.
  • Коротко набирайте замість того, щоб говорити, щоб ізолювати Еві.
  • Після основних оновлень: перезапустіть Evi.


--------------------------------------------------------------------------------
  9. Голосовий вихід (TTS / Piper) — беззвучна або неправильна мова
--------------------------------------------------------------------------------

Симптом: немає звуку, хоча відображається текстова відповідь
Пояснення: відсутній голосовий пакет для мови, неправильний голосовий запис, гучність Windows.

Виправлення:
  • Перевірити piper\ — присутні голоси для вибраної мови?
  • У Evi виберіть мову інтерфейсу з відповідністю голосу, збережіть.
  • Мікшер гучності Windows: Evi не вимкнено?
  • Спробуйте інший вихідний пристрій (навушники, а не монітор).


--------------------------------------------------------------------------------
 10. Мережа, брандмауер, «Блокувати весь трафік»
--------------------------------------------------------------------------------

Симптом: Інтернет, помічники пошти, погода, збій завантажень
Пояснення: глобальне блокування в Evi, брандмауер Windows, корпоративний проксі, VPN.

Виправлення:
  • У Evi вимкніть «Блокувати весь трафік» (або еквівалент), якщо є онлайн-функції
    знаходяться в розшуку.
  • Тимчасово відключіть VPN/брандмауер (лише якщо це дозволено).
  • Корпоративна мережа: попросіть ІТ-спеціалістів надати винятки проксі.


--------------------------------------------------------------------------------
 11. YouTube, медіа, VLC Portable
--------------------------------------------------------------------------------

Симптом: відео не запускається, «відсутній компонент»
Пояснення: VLC Portable відсутній, мережа заблокована, неправильний шлях.

Виправлення:
  • Не видаляйте Portable\VLCPortable (або наданий макет).
  • Дозволити мережу (див. розділ 10).


--------------------------------------------------------------------------------
 12. Мова інтерфейсу користувача, шпаргалки
--------------------------------------------------------------------------------

Симптом: команда не працює, старий текст
Пояснення: шпаргалку відредаговано, але Еві не перезапущено; неправильна назва файлу.

Виправлення:
  • Read cheatsheets\README.txt — один файл на код мови (наприклад, de.txt, en.txt).
  • Після редагування вручну повністю перезапустіть Evi.
  • Зберегти файл як UTF-8 (наголоси).


--------------------------------------------------------------------------------
 13. Безпека: PIN-код і голосове розблокування
--------------------------------------------------------------------------------

Симптом: забув PIN-код, голосове розблокування завжди не вдається
Пояснення: функція безпеки — навмисно важко обійти; кімната занадто шумна.

Виправлення:
  • Тиха кімната, рівень мікрофона, жодних накладень чужих голосів.
  • Відновлення PIN-коду/особистого лише відповідно до політики постачальника — підтримка з підтвердженням.
  • Ніколи не повідомляйте свій PIN-код.


--------------------------------------------------------------------------------
 14. Продуктивність, збої, журнали
--------------------------------------------------------------------------------

Симптом: зависання, раптовий збій
Пояснення: VRAM, RAM, терморегулятор, поганий файл, помилка після оновлення.

Виправлення:
  • Менша модель; закрийте інші програми; перевірити охолодження.
  • Перевірте crash.log або подібний файл у папці Evi, якщо він є.
  • Скасувати останню зміну (видалити нову модель, повернути налаштування).

Симптом: все відчувається мляво
Пояснення: Фонові служби, повний SSD, дуже великий контекст моделі.

Виправлення:
  • Диспетчер завдань: CPU/RAM/GPU; план живлення; вільний простір на SSD (близько 10–15% порожнього).


--------------------------------------------------------------------------------
 15. Якщо нічого не допомагає — інформація на пошту підтримки
--------------------------------------------------------------------------------

  • ID машини (екран активації/інформації)
  • Версія Windows (winver), 64-розрядна
  • Модель GPU + VRAM + версія драйвера
  • Який .gguf (назва файлу, приблизний розмір)
  • Короткі кроки, які ви зробили
  • Точний текст помилки або знімок екрана (не показувати закриті ключі!)
