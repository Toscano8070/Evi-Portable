--------------------------------------------------------------------------------
  クイックチェック (常に最初)
--------------------------------------------------------------------------------

  □ Windows 10 または 11 (64 ビット)、現在のアップデート、大きな変更後に再起動しましたか?
  □ Evi は、解凍途中のアーカイブではなく、完全に解凍されたフォルダーから起動しました。
    クラウド（同期フォルダー）からのみ「ライブ」ではないのですか？
  □ models\llm\ に有効な .gguf が少なくとも 1 つありますか (models\llm\README.txt を参照)。
  □ 十分な空きディスク容量 (モデル + チャット + アップデート) はありますか?
  □ 設定、チートシート、またはモデルを変更した後: Evi を完全に終了します。
    そしてまた始めますか？


--------------------------------------------------------------------------------
  1. オペレーティング システムの基本
--------------------------------------------------------------------------------

症状: インストールまたは起動エラー、「…ではサポートされていません」
説明: Evi は Windows 10/11 のみをターゲットとしています。他のシステムはサポートされていません。

修正:
  • Windows 10 または 11 のみを使用してください。
  • 32 ビット Windows は通常サポートされていないため、64 ビットを使用してください。
  • Windows アップデートをインストールし、PC を再起動します。

症状: アプリをダブルクリックしても何も起こりません
説明: ポリシーがブロックされているか、権限が欠落しているか、ブロックされたパスにランチャーがあります。

修正:
  • 右クリック → 必要な場合のみ管理者として実行 (通常の場合: 標準ユーザー)。
  • ZIP 内から実行しないでください。完全に解凍してください。
  • ウイルス対策プログラムがブロックを記録したかどうかを確認します (セクション 3 を参照)。


--------------------------------------------------------------------------------
  2. フォルダー パス、移植性、クラウドとウイルス対策
--------------------------------------------------------------------------------

症状: ファイルが「消える」、フォルダーが重複する、.gguf が破損する
説明: クラウド同期 (OneDrive、Dropbox など) はファイルをロックしたり、部分的にアップロードしたりする可能性があります
  Evi が書き込み中のデータ。

修正:
  • Evi フォルダーをローカル ドライブに置きます。 C:\EviPortable または D:\Tools\Evi —
    「Documents\OneDrive\…」ではなく、ライブ データに対してアクティブな同期が行われます。
  • Evi ルートにセキュリティ ソフトウェアの除外を追加します (すべての小さなファイルをスキャンします)
    書き込みが遅くなったり、LLM/Piper がひどくロックされたりする可能性があります)。

症状: USB インストール - 非常に遅い、またはエラーが発生する
説明: スティックが遅い、FAT32 サイズ制限、接続不良。

修正:
  • USB 3.x スティック、NTFS、十分なスペース。内蔵SSDを好む。
  • 非常に遅いメディアで巨大なモデルを実行しないでください。より小さい GGUF を使用してください。

症状: 「パスが長すぎます」または特殊文字エラー
説明: Windows MAX_PATH / Unicode — まれですが深いパス + 奇妙な文字
  いくつかの道具を壊す。

修正:
  • 短いパスを使用します。 C:\Evi\、可能であればパスに非 ASCII を含めないでください。


--------------------------------------------------------------------------------
  3. 言語モデル (.gguf) — 応答なし、モデルエラー、ハングアップ
--------------------------------------------------------------------------------

症状: 「AI なし」、空の回答、モデルがロードされない
説明: 予期されたフォルダーに有効な GGUF がありません。ダウンロードが不完全で、間違っています。
  フォーマットされているか、ファイルが破損しています。

修正:
  • モデル\llm\ を開く: 妥当なサイズ (0 バイトではない) の .gguf を少なくとも 1 つ。
  • 再ダウンロード。ブラウザで名前に奇妙な拡張子がないか確認します - 名前を変更します
    ファイルが完了していることが分かっている場合のみ。
  • モデル\llm\README.txt を読む: 推奨される Qwen-2.5 ファイルと VRAM 層。
  • 複数の GGUF: README によれば、複数のファイルが存在する可能性があります。Evi は VRAM に一致するものを選択します。
    疑わしい場合は、既知の正常なファイルを 1 つだけフォルダー内に残しておきます。

症状: GPU があるにもかかわらず、応答が非常に遅い
説明: モデルが GPU → オフロード/CPU に対して大きすぎるか、プリセットが間違っています。

修正:
  • より小さいモデル (例: 14B の代わりに 7B q4_k_m)。
  • サイドバーの GPU プリセットを実際のハードウェアに一致させます。
症状: CUDA / GPU エラー、「メモリ不足」、生成中にクラッシュ
説明: VRAM が不十分です。別のプロセスが GPU を使用しており、ラップトップが iGPU でスタックしています。

修正:
  • NVIDIA ドライバー (nvidia.com) を更新します。
  • ラップトップでは、BIOS/Windows で Evi に「ディスクリート GPU」/「高性能」を強制します。
    利用可能な場合は。
  • Evi で CPU プリセットを試してください (遅いですが安定しています)。
  • GPU を多く使用する他のアプリ (ゲーム、他の AI ツール) を閉じます。


--------------------------------------------------------------------------------
  4. GPU、ドライバー、ノートパソコンのハイブリッド グラフィックス
--------------------------------------------------------------------------------

症状: NVIDIA GPU がインストールされているにもかかわらず、Evi に NVIDIA GPU が表示されない
説明: 間違った電源モード、ドライバーが欠落しており、ハイブリッドは常に iGPU 上にあります。

修正:
  • デバイス マネージャー: NVIDIA にエラー コードはありませんか?
  • 「ハイパフォーマンス」/「アルティメット」モード (ベンダーアプリ: Lenovo Vantage など)。
  • ドライバーの更新後に再起動します。

症状: 画面のちらつき、ドライバーのクラッシュ (TDR)
説明: グラフィックス タイムアウト — 多くの場合、GPU 負荷が高すぎるか、オーバークロックが不安定です。

修正:
  • テスト中はオーバークロックを行わないでください。より小型のモデル。冷却が良くなります。


--------------------------------------------------------------------------------
  5. CPU のみ — 期待と限界
--------------------------------------------------------------------------------

症状：「永遠に時間がかかる」
説明: 7B+ での CPU 推論は GPU よりもはるかに遅いですが、これは正常です。

修正:
  • モデルごとに十分な RAM\llm\README.txt を確保します。
  • バックグラウンド ジョブを閉じます。電源プラン「高パフォーマンス」。


--------------------------------------------------------------------------------
  6. アクティベーション、ライセンス、マシンID
--------------------------------------------------------------------------------

症状: シリアルが受け付けられません
説明: タイプミス、スペース、別のハードウェアのキー、古い ID。

修正:
  • キーを慎重に貼り付けます。先頭/末尾にスペースを入れないでください。
  • メールがこの PC / 有効なマシン ID と一致することを確認します。
  • メインボード/CPU 交換後: 新しい ID — プルーフ チェーンを添えてサポートにお問い合わせください。

症状：メールの返信が来ない
説明: スパムフォルダー、送信元アドレス、件名が間違っています。

修正:
  • スパムをチェックします。件名とマシン ID を明確にして再送信してください。


--------------------------------------------------------------------------------
  7. マイク — レベルなし、Evi には何も聞こえません
--------------------------------------------------------------------------------

症状: キャプチャがありません、メーターが空です
説明: Windows がアクセス、間違った入力デバイス、ケーブル/USB をブロックします。

修正:
  • [設定] → [プライバシー] → [マイク] → デスクトップアプリを許可します。
  • サウンド設定 → 入力: 正しいデフォルト (「スピーチ」) を設定してテストします。
  • Evi の場合: マイクを選択し、保存し、アプリを再起動します。
  • Bluetooth ヘッドセット: ペアリング、バッテリー。遅延が発生する可能性があります。USB マイクを試してください。

症状: マイクが一部のアプリでのみ機能する
説明: 別のアプリの排他モードがデバイスをブロックしています。

修正:
  • 他の通信アプリを閉じるか、「専用マイク」を無効にします。


--------------------------------------------------------------------------------
  8. 音声認識（ローカル）
--------------------------------------------------------------------------------

症状: 音声が認識されず、処理が「停止」する
説明: ウィスパー モデルが欠落または無効、初回ダウンロード、ビジー CPU/GPU。

修正:
  • モデルをフェッチする必要がある場合にのみネットワークを使用します。「models\whisper」を参照してください。
  • Evi を孤立させるには、話す代わりに短く入力します。
  • メジャーアップデート後: Evi を再起動します。


--------------------------------------------------------------------------------
  9. 音声出力 (TTS / Piper) — 無言または間違った言語
--------------------------------------------------------------------------------

症状: テキストの応答が表示されるのに音が出ない
説明: 言語の音声パックがありません。音声入力が間違っています。Windows の音量が異なります。

修正:
  • パイパー\ — 選択した言語の音声が存在するかどうかを確認します。
  • Evi で UI 言語に一致する音声を選択し、保存します。
  • Windows ボリューム ミキサー: Evi がミュートされていませんか?
  • 別の出力デバイス (ヘッドフォンまたはモニタ) を試してください。


--------------------------------------------------------------------------------
 10. ネットワーク、ファイアウォール、「すべてのトラフィックをブロック」
--------------------------------------------------------------------------------

症状: Web、メールヘルパー、天気、ダウンロードが失敗する
説明: Evi、Windows ファイアウォール、企業プロキシ、VPN でのグローバル ブロック。

修正:
  • オンライン機能の場合、Evi で「すべてのトラフィックをブロック」（または同等の機能）をオフにします。
    求められています。
  • VPN/ファイアウォールを一時的に緩和します (許可されている場合のみ)。
  • 企業ネットワーク: プロキシの例外について IT 部門に問い合わせます。


--------------------------------------------------------------------------------
 11. YouTube、メディア、VLCポータブル
--------------------------------------------------------------------------------

症状: ビデオが開始されず、「コンポーネントがありません」
説明: VLC ポータブルが見つからず、ネットワークがブロックされており、パスが間違っています。

修正:
  • Portable\VLCPortable (または提供されたレイアウト) を削除しないでください。
  • ネットワークを許可します (セクション 10 を参照)。


--------------------------------------------------------------------------------
 12. UI言語、チートシート
--------------------------------------------------------------------------------

症状: コマンドが機能せず、テキストが古い
説明: チートシートは編集されましたが、Evi は再起動されませんでした。ファイル名が間違っています。

修正:
  • cheatsheets\README.txt を読みます — 言語コードごとに 1 つのファイル (例: de.txt、en.txt)。
  • 手動編集後、Evi を完全に再起動します。
  • ファイルを UTF-8 (アクセント) として保存します。


--------------------------------------------------------------------------------
 13. セキュリティ: PIN と音声によるロック解除
--------------------------------------------------------------------------------

症状: PIN を忘れ、音声によるロック解除が常に失敗する
説明: セキュリティ機能 — 意図的にバイパスすることが困難です。部屋がうるさすぎる。

修正:
  • 静かな部屋、マイクレベル、他の人の声が重なりません。
  • ベンダー ポリシーに基づく PIN/個人リカバリのみ - 証明付きのサポート。
  • PIN を決して共有しないでください。


--------------------------------------------------------------------------------
 14. パフォーマンス、クラッシュ、ログ
--------------------------------------------------------------------------------

症状：フリーズ、突然のクラッシュ
説明: VRAM、RAM、サーマル スロットル、不正なファイル、アップデート後のバグ。

修正:
  • 小型モデル。他のアプリを閉じます。冷却を確認します。
  • Evi フォルダに crash.log などがある場合は、それを確認します。
  • 最後の変更を元に戻します (新しいモデルを削除し、設定を元に戻します)。

症状：すべてが鈍く感じられる
説明: バックグラウンド サービス、フル SSD、非常に大規模なモデル コンテキスト。

修正:
  • タスクマネージャー: CPU/RAM/GPU。電源プラン。 SSD の空き容量 (約 10 ～ 15% の空き)。

 15. 何も解決しない場合 — サポートメールの情報
--------------------------------------------------------------------------------

  • マシンID (アクティベーション/情報画面)
  • Windows バージョン (winver)、64 ビット
  • GPU モデル + VRAM + ドライバーのバージョン
  • どの .gguf (ファイル名、おおよそのサイズ)
  • 実行した短い手順
  • 正確なエラー テキストまたはスクリーンショット (秘密キーは表示しないでください)
